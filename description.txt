Descrição do programa:
Sistema de gerenciamento de programas para fácil instalação ou desinstalação numa possível formatação ou configuração de computador ou numa otimização.


Descrição do site:
Site que receberá os dados do arquivo log.log gerado pela execução do programa. Quando o programa é executado, ele abre uma aba no navegador com este site. Este projeto irá monitorar e retornar ao usuário os dados do arquivo em tempo real para que o usuário possa saber o progresso do programa.
Esse site será salvo no Github e terá hospedagem vinculada na Vercel. Qualquer coisa que precisar, poderá utilizar tanto o Github Actions quanto o Back-end da Vercel para que algumas funcionalidades funcionem corretamente.


FRONT-END programa:
- Layout Language: EN-US;
- Primeira tela:
    - Conterá o título variável do programas, que varia de acordo com o Sistema Operacional;
    - Container onde aparecerá cada classe de função em forma de checklist
    - No canto inferior esquerdo terá um botão que ao clicar seleciona ou desmarca todas as opções do container principal;
    - No canto inferior direito, haverá um botão de "Next", que ao clicar nele o sistema fechará a primeira tela e abrirá a segunda tela, e o sistema transmitirá um array somente com os dados das opções selecionadas;
- Segunda tela:
    - Quando for aberta a segunda tela, o sistema irá obter do link https://raw.githubusercontent.com/JLBBARCO/programs-manager/main/system/ os JSONs referentes ao array retornado pela primeira tela, e mostrará esses dados no container principal;
    - Container principal:
        - Menu suspenso: cada opção é referente ao array recebido da primeira tela, e ao clicar numa opção específica, o programa mostrará os dados do JSON referente, organizado em ordem alfabética;
        - Conteúdo: será mostrado um checklist dos dados do JSON referente à opção seleciona no menu suspenso;
    - No canto inferior esquerdo, haverá um container com dois botões:
        - Na direita, terá um botão de "Add Program". Quando clicado, ele abre uma tela com as seguintes características:
            - No topo terá uma barra de pesquisa e um botão de "Search";
            - Quando o usuário digitar algo na barra e clicar em search, o sistema rodará uma função do backend que lista os programas relacionados encontrados;
            - No container principal será mostrado os dados desse array, mas em três colunas. A primeira será um botão, que alterna entre "Add" e "Remove" dependendo do click do usuário. Na segunda coluna será mostrado o nome do programa, e na terceira coluna será mostrado o id do programa;
            - No canto inferior direito terá um botão "Submit" que fechará a tela;
        - Na esquerda do container, terá outro botão chamado "Remove Programs", que quando clicado fará:
            - Abrirá uma tela de ideia semelhante a do "Add Program";
    - No canto inferior direito, haverá um botão de "RUN", que fechará a tela e dará início ao sistema principal em segundo plano;


FRONT-END site:
- Layout Language: EN-US;
- Na primeira tela, será mostrado três contêineres com os dados obtidos do Back-end com as seguintes informações:
    - Cada contêiner conterá um tamanho fixo, com uma barra de rolagem horizontal. Os dados serão mostrados de forma que os mais antigos aparecerão acima e os mais recentes aparecerão abaixo. O contêiner terá rolagem automática para enfatizar os últimos dados, mas se o usuário rolar para cima manualmente, a rolagem automática é pausada até o usuário rolar novamente ao fim dos dados;
    - Os dados serão recebidos da seguinte forma:
        - [01/01/2000 00:00:00] [INFO]
        - [01/01/2000 00:00:00] [SUCCESS]
        - [01/01/2000 00:00:00] [WARNING]
        - [01/01/2000 00:00:00] [ERROR]
    - Os contêineres serão organizados de acordo com os resultados {INFO, WARNING, ERROR} e mostrarão somente os dados referentes;
    - O dado que será recebido será do tipo: [01/06/2026 12:30:45] [SUCCESS] Visual Studio Code installed; E o que será mostrado para o usuário será da seguinte forma: 01/06/2026 12:30:45 | Visual Studio installed; Com a data e hora na cor #808080 e a informação na cor padrão de texto do site;
- Abaixo, a segunda tela só aparecerá se o arquivo possuir um histórico extenso.
- Esse histórico será definido pelo cálculo de tolerância de 1 minuto de atraso a partir da abertura do site. Ex.: se no arquivo possuir a datação [01/06/2026 00:00:00] e o site possuir a datação [01/06/2026 00:01:00], a informação da linha será mostrada na tela principal. Agora, se a datação do arquivo for a mesma, mas do site for [01/06/2026 00:01:01], essa informação será mostrada na tela secundária onde será a partição de histórico de execuções antigas. Esse sistema de filtro servirá para que o site de ênfase aos dados atuais, mas se o usuário quiser ver os dados de outras execuções ele consegue. O tempo de 1 minuto de tolerância a partir da execução do programa em relação a datação do site é para que se o site demorar para carregar, os dados da execução atual não irão para o histórico geral.
- Se o time hate for expedido, ao invés de aparecer a tela principal e secundária descritas acima, será mostrada uma mensagem dizendo que a porta 8000 não foi aberta ou o arquivo não foi compartilhado, e pede ao usuário que faça refresh na página ou reexecute o programa. Logo abaixo será mostrado um botão que levará ao repositório https://github.com/JLBBARCO/programs-manager ;
- No rodapé, terá um card que carregará as informações do JSON obtido do repositório de portfólio pelo Back-end, utilizará o dado "url" para ser o href do link, o "name" será utilizado na sub-tag title do link, e o "iconName" será utilizado para dizer qual ícone será inserido dentro do link. A organização do container será em display flex, flex-direction row;


BACK-END programa:
- Quando o usuário clicar em "Add Program" na tela secundária, o sistema irá abrir a 3.1ª tela descrita na parte de frontend. O sistema da barra de pesquisa pesquisará no sistema de gerenciador de pacotes, ex.: WinGet no Windows, e o botão que alterna entre "Add" e "Remove" adicionará o nome e id do programa referente à um array que, ao clicado, salvará esses dados num array, e após clicado em "Submit", será salvo em um JSON interno chamado user_install.json. Quando o sistema for rodar somente em segundo plano, ele lerá o user_install.json e executará um sistema que utiliza por ex.: "winget install -id <id-do-programa>" no Windows, para instalar determinado programa;
- Quando o usuário clicar em "Remove Program" também da tela secundária, essa 3.2ª tela terá um funcionamento muito parecido do sistema do botão "Add Program". A diferença é que enquanto a tela estiver carregando, o sistema lista e mostra todos os programas instalados e salvará o array no arquivo user_uninstall.json, para ser utilizado pelo gerenciador de pacotes, ex.: "winget uninstall -id <id-do-programa>";
- Quando o usuário clicar em "Run" na última tela, todas as telas serão fechadas e o programa será executado somente em segundo plano. Simultaneamente ao fechamento, ele salvará no arquivo log.log a seguinte mensagem: [dd/mm/aaaa hh:mm:ss] [INFO] Start system . Também abrirá o compartilhamento do arquivo log.log pela porta 8000 para o site https://programs-manager-website.vercel.app e abrirá esse site;
- Logo após as primeiras execuções, o programa iniciará as tarefas principais na seguinte ordem:
    - Atualizar o instalador de pacotes, exemplo Windows: Microsoft.AppInstaller;
    - Se o user_uninstall.json existir, faça a desinstalação dos programas listados neste arquivo;
    - Se o user_install.json existir, faça a instalação dos programas listados nele;
    - Instalar ou atualizar se possível os programas listados nos seguintes arquivos JSON:
        - drivers
        - essentials
        - screen
        - ti_tools
        - development
        - server
        - games
        - customizations
        Praticamente todos os arquivos JSON possuem a seguinte estrutura de dados:
            {
                "name": "<nome-do-programa>",
                "id": "<id-do-programa>"
            }
        O único que possui dados variados é o customization.json:
            Ele pode ter tanto essa estrutura:
                {
                    "name": "<nome-do-programa>",
                    "id": "<id-do-programa>"
                }
            Quanto essa estrutura:
                {
                    "name": "<nome-da-função>",
                    "function": "<id-da-função>"
                }
            Esse objeto chamado function possui um id que ao ser verificado pelo sistema, executará determinada função. Ex.:
                {
                    "name": "Dark Mode",
                    "function": "dark-mode"
                }
            Esse array significa que se ele for executado, deverá aplicar o modo escuro no sistema operacional.
- Ao escrever no log.log, o sistema terá quatro tópicos de mensagens:
    - [INFO]
    - [SUCCESS]
    - [WARNING]
    - [ERROR]
    O tópico info será para informar que tal coisa começou ou aconteceu. Exemplos: "Start system", "Find and read https://raw.githubusercontent.com/JLBBARCO/programs-manager/main/system/windows/install/essentials.json", "End system";
    Já o tópico success servirá para informar que algo foi bem sucedido. Exemplos: "Port open", "Visual Studio Code installed";
    O warning por sua vez, servirá para indicar que algo aconteceu parcialmente correto. Ex.: "Find and not read https://raw.githubusercontent.com/JLBBARCO/programs-manager/main/system/windows/install/essentials.json", "Visual Studio Code updated";
    O error servirá para algo que deu completamente errado. Ex.: "Not found https://raw.githubusercontent.com/JLBBARCO/programs-manager/main/system/windows/install/essentials.json", "Visual Studio Code not installed and not updated";
- O sistema lerá os seguintes arquivos no caminho https://raw.githubusercontent.com/JLBBARCO/programs-manager/main/system/<sistema operacional>/install/:
    - customization.json
    - development.json
    - drivers.json
    - essentials.json
    - games.json
    - screen.json
    - server.json
    - ti_tools.json
- Ele lerá também o restante dos arquivos no caminho https://raw.githubusercontent.com/JLBBARCO/programs-manager/main/system/<sistema operacional>/custom/:
    - vision-cursor-black/
    - list_startup_programs.py
    - listar-startup.bat
    - white_list.txt
    - WHITELIST_README.md


Github Actions:
- O Github Actions deverá compilar o programa em cada sistema operacional. O Actions deverá utilizar os arquivos "build.bat", build.sh" e "build-mac.sh" para compilar para cada sistema operacional. Essa automação deverá armazenar a compilação em um arquivo de compactação, ex.: zip, e armazenará em um Github Release novo, mas seguindo algumas regras:
    - Se a branch do commit for a main, o Actions deverá salvar os compilados como Last Release;
    - Já se a branch for a beta, o Actions deverá salvar como Pre-release;
- A imagem src/assets/icon/icon.ico será utilizada como ícone do programa;
- A pasta system não precisa ser incluída na compilação pois todos os arquivos contidos nela serão obtidos via Github RAW;
- A automação também deverá gerar um print de cada sistema operacional utilizando o bot do Github. Esses prints serão em formato WEBP, e eles substituirão os arquivos dos caminhos:
    src/assets/img/macos.webp
    src/assets/img/linux.webp
    src/assets/img/windows.webp
- Depois dos prints separados, será montada uma imagem com os três prints lado a lado, e substituirão o arquivo do caminho:
    src/assets/img/thumbnail.webp


Github RAW:
- Os arquivos "./run.ps1" e "./run.sh" serão utilizados para baixar e rodar o sistema diretamente do terminal, tanto Windows quanto Linux quanto MacOS, baixando o compilado referente ao sistema operacional diretamente da Last Release. Os comandos utilizados serão:
    "irm https://raw.githubusercontent.com/JLBBARCO/programs-manager/main/run.ps1 | iex" para Windows;
    "curl -fsSL https://raw.githubusercontent.com/JLBBARCO/programs-manager/main/run.sh | bash" para Linux
    "curl -fsSL https://raw.githubusercontent.com/JLBBARCO/programs-manager/main/run.sh | bash" para MacOS
Ao rodar o link https://raw.githubusercontent.com/JLBBARCO/programs-manager/main/run.ps1 no terminal, o sistema deverá obter o compilado da Latest Release. Mas se o usuário rodar o link https://raw.githubusercontent.com/JLBBARCO/programs-manager/beta/run.ps1 o sistema deverá rodar o Pre-release


BACK-END site:
- A cada vez que o site for aberto, ele armazenará temporariamente a data e hora de loading para ser utilizado pelos sistemas de filtros;
- Quando o site for carregado, ele irá monitorar a porta 8000 por 30 segundos. Se ela não for aberta ou não for compartilhado o arquivo log.log por ela, o site irá parar de monitorar ela até que o usuário faça um refresh;
- O sistema de filtro por datação será com base na seguinte informação com data e hora de exemplo:
    [01/01/2000 00:00:00] [INFO] Start system
- O sistema só irá parar de monitorar o arquivo quando a informação mais recente for:
    [01/01/2000 00:00:00] [INFO] End system
- O sistema irá ler o https://raw.githubusercontent.com/JLBBARCO/portfolio/main/src/json/areas/contact.json e inserirá os dados no container de contato do rodapé. A requisição desse json será feita pela Vercel a cada hora para que não tenha muitas requisições no github e que o site fique leve para o usuário;
